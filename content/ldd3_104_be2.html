BE2 - Fonctions définies par l’utilisateur#

Rappels sur les fonctions#

Définition d’une fonction (def)#

la syntaxe de la définition d’une fonction est

où

  • <nom_de_la_fonction> est l’identifiant de la fonction dans l’espace de nom.

  • <paramètres> est une suite, éventuellement vide, de paramètres de la fonction.

  • <suite> est le corps de la fonction, il s’agit d’un bloc d’instructions qu’elle exécutera lorsqu’elle sera appelée ; ce bloc doit être non vide et indenté.

Appel à une fonction#

La syntaxe de l’appel à une fonction est

où

  • <nom_de_la_fonction> est l’identifiant de la fonction dans l’espace de nom.

  • <arguments> est une suite, éventuellement vide, d’argument à transmettre à la fonction.

Objet retourné par une fonction (return)#

L’appel à une fonction renvoie un objet qui est déterminé par la manière dont la fonction se termine.

  • si la fonction exécute l’instruction return, alors elle se termine et renvoie l’objet à la droite du mot-clé return
    Exemple : return 3 renvoie l’entier 3, return [1, 2] renvoie la liste [1, 2] et return 1, "b", [4, 5] renvoie le n-uplet (1, "b", [4, 5]) (identique à return (1, "b", [4, 5])). L’instruction return peut apparaitre plusieurs fois dans u

  • si l’exécution des instructions du bloc <suite> arrive à la fin du bloc, alors elle se termine et renvoie l’objet None

Exemple 1 : cas particulier sans paramètres et sans instruction return

# Définition de la fonction
def affiche_bonjour():
    """Affiche le message 'Bonjour'."""
    print("Bonjour")
    
    
# Appel de la fonction sans argument
affiche_bonjour()

Bonjour

Exemple 2 : avec deux paramètres et une instruction return

# Définition de la fonction
def salutation(nom, prenom):
    """Construit une salutation personnalisée."""
    return " ".join(["Bonjour", prenom, nom])


# Appel de la fonction avec les arguments "Lovelace" et "Ada",
# le résultat est assigné au nom "resultat"
resultat = salutation("Lovelace", "Ada")

# Appel de la fonction avec les arguments "Lovelace" et "Ada",
# le résultat n'est pas assigné à un nom
salutation("Lovelace", "Ada")

'Bonjour Ada Lovelace'

2. Fonctions récursives#

Le processus dans lequel une fonction appelle elle-même directement ou indirectement est appelée récursion et la fonction correspondante est appelée une fonction récursive.

En exemple classique est la fonction factorielle. Sa définition mathématique itérative

\[ n! = \prod_{k=1}^{n} k = 1 \times 2 \times 3 \times \cdots \times n \]

donne lieu à un codage itératif

def fact_iter(n):
    """Calculer n! de manière itérative."""

    resultat = 1
    for i in range(1, n + 1):
        resultat = resultat * i
    return resultat


fact_iter(10)

3628800

Mais la fonction factorielle peut également être définie mathématiquement de façon récursive. Pour tout entier naturel \(n\), on a :

\[\begin{split} n! = \begin{cases} 1 & \text{si } n = 0, \\ n \times (n-1)! & \text{si } n > 0. \end{cases} \end{split}\]

La fonction factorielle est utilisée pour se définir elle-même. Le codage associé est

def fact_rec(n):
    """Calculer n! de manière récursive."""

    if n == 0:
        return 1
    return n * fact_rec(n - 1)


fact_rec(10)

3628800

2.2 ⚙️ Exercice - Tours de Hanoï#

Le problème des tours de Hanoï est un jeu de réflexion consistant à déplacer des disques de diamètres différents d’une tour de « départ » à une tour d’« arrivée » en passant par une tour « intermédiaire » et ceci en un minimum de coups, tout en respectant les règles suivantes :

  • seul un disque est déplacé à la fois,

  • un disque ne peut être posé que sur un emplacement vide ou un disque de taille plus importante.

Travail demandé : Le programme recherché doit établir la liste des 127 mouvements à faire pour déplacer une tour de Hanoï composée de 7 disques aux couleurs de l’arc-en-ciel (Violet, Indigo, Bleu, Vert, Jaune, Orange, Rouge).

Chaque mouvement devra être défini par son numéro d’ordre, la couleur du disque à déplacer, le nom de la tour où le disque est pris et le nom de la tour où le disque est déposé.

Coup  26  :  Orange  de  A  à  B

Modélisation du problème

Un algorithme récursif est un algorithme qui résout un problème en calculant des solutions d’instances plus petites du même problème [wikipedia].

Pour mettre en place un programme récursif dans le cas des tours de Hanoï, il faut exprimé la résolution du problème des tours à \(n\) disques à partir du problème des tours à \(n-1\) disques. La figure suivante, illustre le principe de la récursion pour \(n=7\).

Les paramètres d’entrée de la fonction hanoi() sont :

  • \(n\) nombre de disques à déplacer,

  • \(D\) empilement de disques de la tour de départ,

  • \(A\) empilement de disques de la tour d’arrivée,

  • \(I\) empilement de disques de la tour intermédiaire.

La valeur retournée par la fonction est le triplet (\(D\), \(A\) et \(I\))

L’algorithme en pseudocode de la fonction récurrente est :

fonction hanoi(n, D, A, I)
    si n > 0
        hanoi(n-1, D, I, A)
        dépiler le disque au sommet D pour l'empiler sur A 
        hanoi(n-1, I, A, D)
    retourner (D, A, I)

Conseils

Vous pouvez résoudre progressivement le problème

  1. coder une 1re version sans affichage, mais qui trie correctement des tours

  2. coder une 2e version qui affiche la couleur du disque déplacé à chaque coup

  3. coder une 3e version qui affiche la tour de départ et d’arrivée à chaque coup

  4. coder la version finale qui affiche le numéro de chaque coup.
